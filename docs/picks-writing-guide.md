# 选股.md 写入指导

本文用于统一后续向 [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 写入内容时的格式、顺序与分析口径。

目标不是让 `选股.md` 变成技术结果原始转储，而是让它成为“基于当日 `watchlist`、结合主线与主观判断后的最终整理记录”。

当前每日筛选已切换为 `Phase1/Phase2/Phase4/Phase5/Phase7/Phase8 + pattern` 口径。日常运行 `daily-screening` 后，主 `watchlist` 有三条入口：pattern 命中票不再按 Phase1/Phase2 分数过滤，只要求 Phase4 分数 `> 70`；Phase4 Top 补充票仍先用 Phase1 和 Phase2 各自排除最高风险 20%，再按 `centered_risk_score` 补入主候选；Phase8 Top5 作为“三个交易日内涨停机会且控制大跌风险”的短线观察补充，独立展示，不参与主排序。所有入口都会在最终入选前排除当日涨幅超过 `9.9%` 的股票。`daily-screening` 还会生成独立的 `intraday_pool_YYYY-MM-DD`，供下一交易日盘中快速扫描；它不是正式选股名单。Phase8 高分只说明短线弹性值得看，不替代主线、P1/P2/P4 和技术结构判断。Phase7 的 `trade_permission` 是日期级买点环境判断，必须写在每天最上方；如果它是 `no_trade`，当日候选只作为观察池，不作为积极买入名单。Phase5 是长周期极端风险画像，只作为风险提示字段，不作为日内买点信号。旧口径里若写成 `pattern4`、`pattern5`、`pattern7`，写入 [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 时统一按 Phase4、Phase5、Phase7 字段处理。

## 使用范围

本指导适用于所有需要新增或更新 [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 的场景。

写入时必须遵守以下前提：

1. 基于当日主 `watchlist` 进行选股
2. 读取 [track_stock.xlsx](/C:/Users/wdyab/Desktop/wdy/stocks/track_stock.xlsx) 的 `Sheet2`，对手动跟踪股票逐只给出是否适合买入的结论
3. 结合 [主线.md](/C:/Users/wdyab/Desktop/wdy/stocks/主线.md)，优先选择博主给出的主线对应个股
4. [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 最前面的“人工精选核对清单（固定）”必须保留，不得覆盖、删除或移动
5. 新日期的结果写在固定清单之后、已有日期之前，旧日期的结果保留在文件后面
6. 一个日期内，先给出最终选股结果表，再给出 Phase8 Top5 短线观察表，再给出交易辅助信息表，再给出跟踪股票总结，最后给出简要分析

如果 [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 中已经有对应日期的选股列表，必须先删除该日期已有列表并完全忽略旧内容，只根据最新筛选结果、最新主线和本指导重新列出该日期选股列表。

其中主 `watchlist` 是“Phase4 分数合格的 pattern 命中 + 通过 Phase1/Phase2 硬过滤后的 centered risk 补充”的正式盘后候选池，不等同于最终买入名单。`intraday_pool_YYYY-MM-DD` 是次日盘中快速扫描池，不能替代主 `watchlist` 写入 [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md)。`watchlist_pattern` 保留为兼容输出，不作为最终写作入口。

另外，写入表格时应尽量排除明显英文状态值：

- `模式`、`买入`、`强力买入`、`突破`、`回踩` 等优先使用中文
- `ATR14`、`ATR%`、`MACD` 等通用缩写可保留

## 日中名单简洁版要求

[选股-日中.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股-日中.md) 的目标是快速决策，不写成长复盘。写入时必须尽量短，让人能一眼看到什么可买、什么只能观察、什么不能追。

日中选股表固定使用下面的压缩字段：

- `股票代码`
- `股票名称`
- `主线`
- `P1/P2/P4`
- `P4五日均/std`
- `P8`
- `盘中涨幅`
- `建议总仓位`
- `技术/前日形态`

日中表格规则：

- 不显示 `intraday_pool_score`
- `P1/P2/P4` 放在同一列，写成 `80 / 78 / 86`
- `P4五日均/std` 单独一列，紧跟在 `P1/P2/P4` 后面，写成 `88 / 5`
- `P8` 单独一列，紧跟在 `P4五日均/std` 后面；如果 Phase8 模型尚未训练完成或当天未生成预测，写 `缺失`
- 表格中不要放长篇 `日中判断`
- `日中判断` 放在表格下方，用短文本逐只说明
- 不需要写单独的 `主线判断`
- 不需要写单独的 `盘中状态`
- 不需要写 `不选或降级`

日中判断写法只保留结论和关键原因，例如：

```md
日中判断：

- 000000示例股票：可观察。属于当前主线，P1/P2 接近 80，涨幅未过热，尾盘重点看能否站稳关键位。
- 000001示例股票：不追。盘中涨幅偏高，即使 P4 高，也要等回落后确认。
```

手动跟踪股票仍要单独总结，但同样使用短表格和短判断，不写成长段复盘。

## 总体原则

### 1. `选股.md` 不是候选池原样转储

主 `watchlist` 是技术候选池，`选股.md` 是基于候选池进一步整理后的结果。

因此写入时：

- 不要求把全部候选都写进去
- 可以只保留少量更符合主线和当日判断的标的
- 不要把纯技术强度但明显脱离当前主线的个股全部塞进最终记录
- Phase4 分数必须展示。对 `phase4_top` 补充票，Phase4 是入表核心依据，但排序采用 `centered_risk_score`，即 P1/P2 越接近 80 越加分；对 pattern 命中票，Phase4 是唯一自动入池分数门槛，必须 `> 70`，P1/P2 只作为风险解释，不作为剔除条件。

### 2. 先写 Phase7 交易许可

每天日期标题下面先写一行：

```md
Phase7交易许可：允许 / 不建议
```

如果 `watchlist.phase7_trade_permission = no_trade` 或 `watchlist.trade_permission = no_trade`，应写成：

```md
Phase7交易许可：不建议。Phase7 判断当前属于最高风险 20% 交易日，以下仅作为观察池，不作为积极买入名单。
```

如果是 `allow`，可以写成：

```md
Phase7交易许可：允许。仍需结合主线、量价、压力位和盘中状态复核。
```

如果 `watchlist` 同时带有 `phase7_score_100` 和 `phase7_feature_trade_date`，许可行后可以补一行：

```md
Phase7交易日分：100，特征日期 2026-05-06。
```

这里的“特征日期”对应 `phase7_feature_trade_date`；当目标日没有最新日线时，系统会使用不晚于目标日的最新本地特征日。

### 3. 主线优先

选股时要先看 [主线.md](/C:/Users/wdyab/Desktop/wdy/stocks/主线.md)。

优先顺序建议如下：

1. 博主明确指出的第一梯队主线
2. 明确指出的第二梯队主线
3. 轮动与预期线
4. 短线事件题材
5. 无法归入主线、只有技术形态支撑的个股

如果某只股票技术面很好，但不属于当前主线，应降低优先级。

### 4. 结果先于分析

一个日期段落内部，必须先给出选股结果，再给出分析。

顺序固定为：

1. 日期标题
2. Phase7交易许可
3. 最终选股结果表
4. Phase8 Top5 短线观察表
5. 交易辅助信息表
6. 跟踪股票总结
7. 逐股口语化说明
8. 当日市场情绪
9. 主线变更
10. 值得注意的股
11. 如有必要，再补充简短备注

### 字段解释

`优先级与来源`

- 最终选股结果表不再单列 `梯队` 和 `来源`
- 优先级通过主表从上到下的排序体现，越靠上优先级越高
- 候选来源统一来自主 `watchlist`

`连续上榜天数`

- 指截止当日，该股票连续出现在主 `watchlist_YYYY-MM-DD.json` 的天数
- 必须按主 `watchlist` 计算，不按其他历史候选文件单独计算
- 中间只要断一天，下一次重新从 `1` 开始
- 这是持续强度参考，不单独决定是否推荐，但同等条件下连续天数更高的票可优先

`P1/P2/P4`

- 对应 `watchlist` 或 `track_stock.xlsx` 中的 `phase1_score_100 / phase2_score_100 / phase4_score_100`
- 主表必须把三项放在同一列，写成 `P1 / P2 / P4`，例如 `72 / 66 / 86`
- P1/P2 是 0-100 买入友好风险分，低分需要警惕，但不是越高越好；当前回测更偏好 P1/P2 在 `70-90`，尤其接近 `80` 的状态
- P4 是 Phase4 Alpha158/Qlib 收益模型的 0-100 横截面展示分，数值越高代表模型给出的短期收益预期越靠前
- P4 只能作为同一主线、同等风险过滤后的排序参考，不能单独构成推荐理由
- 如果某项字段缺失，该项写 `缺失`，不要用旧模型结果或外部技术评分替代

`P4五日均/std`

- 对应 `phase4_5d_mean / phase4_5d_std`
- 主表和跟踪表都必须展示，写成 `均值 / 标准差`，例如 `88 / 5`
- 五日 P4 均值衡量 Phase4 收益排序的连续性；标准差衡量该连续强度是否稳定
- 同一主线内优先选择主 `watchlist` 中同时进入五日 P4 均分 Top5 的股票；如果当日 P4 很高但五日均值低、std 高，要写成“一日跳升，稳定性不足”

`P8`

- 对应 `phase8_score_100`
- P8 是三日内触及涨停机会模型的 0-100 展示分，训练目标已经惩罚三日大跌风险
- P8 目前只独立展示，不参与主 `watchlist` 的 centered 排序，也不替代 Phase4 的中期收益模型
- 主表、跟踪表和 Phase8 Top5 表都必须展示 P8；如果当天没有 P8 预测文件，写 `缺失`
- P8 高分可以作为短线弹性提示，但不能单独构成推荐理由；必须同时看当日涨幅、主线、P1/P2 风险分、P4 和 ATR 仓位

`Phase5`

- 对应 `watchlist` 或 `track_stock.xlsx` 中的 `phase5_score_100`
- 这是周频/年度极端风险画像，不是每日买入信号
- 主表里必须简写为可读状态：
  - `稳健`：`phase5_score_100 >= 70`
  - `中性`：`40 <= phase5_score_100 < 70`
  - `偏弱`：`phase5_score_100 < 40`
  - `缺失`：没有 Phase5 字段
- 逐股说明里只在分数偏弱时展开，不要把 `NCSKEW/DUVOL` 原始数值塞进主表

`行业/主线`

- 优先填写与 [主线.md](/C:/Users/wdyab/Desktop/wdy/stocks/主线.md) 对应的主线名称
- 如果确实无法归类，再写 `未分类`

`符合模式/背离`

- 写清模式编号
- 如果存在顶背离、底背离、MACD 背离、量价背离，要明确写出来
- 示例：
  - `模式1`
  - `模式2 + 底背离`
  - `模式1 + 顶背离`
  - `模式3 + MACD底背离 + 量价看多`

`当日涨幅`

- 对应 `watchlist` 中的 `涨幅%` 或 `daily_return_1d`
- 主表必须展示，用来直接判断是否已经过热
- 当前主 `watchlist` 已在最终入选前排除当日涨幅 `> 9.9%` 的股票；如果看到接近 `9.9%` 的候选，推荐理由里要提示不要追高

`P1/P2 风险解释`

- 对应 `phase1_score_100 / phase2_score_100`
- 建议写成 `Phase1 / Phase2` 的形式，例如 `72 / 66`
- 两个分数都是 0-100 买入友好分，低分需要警惕，但不是越高越好。当前回测更偏好 P1/P2 在 `70-90`，尤其接近 `80` 的状态。
- 已进入主 `watchlist` 的 Phase4 Top 补充票已经通过 Phase1 和 Phase2 的最高风险 20% 硬过滤；pattern 命中票可能没有通过 P1/P2 硬过滤，因为部分形态天然波动更高
- 最终进入“最终选股结果表”的股票，P1 和 P2 不应太低，但门槛按来源分层：
  - `phase4_top` 补充票：按代码口径至少满足 P1 `>= 40`、P2 `>= 50`、P4 `>= 70`，并优先选择 P1/P2 接近 `80` 的票。
  - pattern 命中票：取消 P1/P2 分数底线，只要求 P4 `> 70`；如果 P1/P2 很低，要在判断里明确写出“形态高波动、风险分偏低”，不能把它当成低风险票。
  - 非 pattern 来源只要 P1 或 P2 低于自动门槛，默认不进最终选股结果表。
- 同主线内仍应优先选择 P1/P2 处在较优区间、止损成本更清晰的票。不要因为 P1/P2 都超过 `90` 就自动升为最高优先级。

### 模式回测结论与持有周期参考

模式 1-6 的纯模式回测结果只说明“形态本身”的基础表现，不叠加 `watchlist`、主线、MACD/量价等后续过滤条件。当前结果显示，所有模式的裸信号胜率都低于 50%，因此写 [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 时不能把“命中模式”直接写成确定性买点，只能写成候选结构和观察窗口。

| 模式 | 更适合的观察周期 | 写作口径 |

| 模式2 | 20 日观察，但不宜单独作为买点 | 平均最大浮盈最高，但回撤也大、胜率偏低；写作时必须强调“突破位能否站稳”，不能只因放量突破就推荐。 |
| 模式3 | 20-40 日补充观察 | 适合写成“突破后的缩量回踩 / 二次上车观察”，但稳定性不如模式1、模式5、模式6，需要看 MA20 和突破位承接。 |
| 模式4 | 5-10 日试错观察 | 新口径是老鸭头鸭鼻孔金叉，属于鸭嘴张开前的低吸观察点；只有主线强、回调缩量充分、MA5/MA10 金叉不久且未大幅突破鸭头顶时才考虑进主表。 |

综合排序用于同等主线条件下的技术优先级参考：

`模式5 > 模式1 > 模式6 > 模式3 > 模式2 > 模式4`

这个排序不是机械买入顺序。若某只低排名模式股票属于当日核心主线、同时有量价看多或底背离，可以排到更前；若某只高排名模式股票脱离主线、出现顶背离或量价看空，应降级。

- 模式4当前是老鸭头鸭鼻孔试错点，不应写成突破确认，除非主线和技术过滤同时很强。

`模式1 > 模式2 > 模式3 > 模式5 > 模式4 > 模式6`

`收盘价`

- 对应 `watchlist` 中的 `close`、`收盘价` 或 ATR 表里的最新收盘价
- 只放在交易辅助信息表，不放在最终选股结果表
- 用于核对 ATR 止损、止盈和建议总仓位是否合理

`ATR14`

- 对应 `watchlist` 中的 `ATR14`
- 用于描述个股的日线平均波动幅度
- 建议保留 1 位到 2 位小数

`ATR%`

- 对应 `watchlist` 中的 `ATR%`
- 用于横向比较不同股票的相对波动率
- 必须写成百分比形式

`建议总仓位`

- 对应 `watchlist`、`watchlist_YYYY-MM-DD.csv`、`track_stock.xlsx` 中的 `建议总仓位%`
- 这是按“单笔总最大亏损 2%、`D = 2ATR14`、第二批回撤 `D/2`、第三次加仓前有效平均止损距离 `0.85D`、单一标的最高 40%”推导出的计划满仓比例，不是第一批买入比例
- 公式：`建议总仓位% = min(40, 100 * 0.02 * 当前价 / (0.85 * 2 * ATR14))`
- 如果 `ATR%` 用百分数数值表示，例如 `4` 表示 `4%`，也可写成 `min(40, 117.65 / ATR%)`；例如 `ATR% = 4`，建议总仓位约 `29.4%`
- 写入表格时保留 1 位到 2 位小数；数值明显低于 `20%` 时，说明波动较大，应降低实际买入优先级或缩小首批仓位

`建议止损`

- 必须填写
- 固定写 `2倍ATR` 对应的具体价格，不再加 `2ATR止损参考` 这类文字前缀
- 示例：`18.30`

`建议止盈`

- 必须填写
- 固定写 `2倍ATR` 对应的具体价格，不再加 `2ATR止盈参考` 这类文字前缀
- 示例：`21.70`

`趋势/触发补充`

- 日常主流程不再接入 `trend`，这一列只写主 `watchlist` 已经带出的技术状态
- 优先写这些内容：
  - `macd_cross_state`，如 `金叉 / 死叉 / 上方运行 / 下方运行`
  - `macd_divergence_state`，如 `MACD顶背离 / MACD底背离`
  - `volume_price_divergence_state`，如 `量价看多 / 量价看空`
- 这一列是“辅助判断”，不要求把全部字段原样抄进去
- 示例：
  - `突破 + 金叉`
  - `回踩 + MACD底背离`
  - `突破 + 量价看多`

## 选股规则

### 1. 必须基于当日候选池

写入 [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 时，候选来源必须来自当日主 `watchlist`，而不是随意从全市场结果中重挑。

- 不要从 `reports/full_market_model` 里手工捞出未进入主 `watchlist` 的股票；Phase 预测文件是构建候选池的依据，不是写作时重新选股的入口

### 2. 主线优先于纯技术

当主线和技术面冲突时，优先考虑主线一致性。

推荐选择逻辑：

1. 先找属于当日主线的主 `watchlist` 个股
2. 先按来源检查：`phase4_top` 补充票必须看 `phase1_score_100 / phase2_score_100` 是否太低；pattern 命中票不因 P1/P2 低分直接剔除，但必须写清波动和止损风险
3. 同主线内再比较止损成本和模式结构完整度
4. 再看 `phase4_score_100`、底背离、金叉或量价看多
5. 暂时不考虑非主线补位；如果主线内没有合适股票，可以少选或不选

### 3. 可自动筛选的技术加减分

下面这些规则只使用当日 `watchlist` / `patterns_all` 已有字段，可以在人工写入前先做自动排序或剔除。它们不是替代主线判断，而是帮助把同一主线内的候选分出优先级。

通用优先项：

- `phase1_score_100`、`phase2_score_100` 不弱：对 `phase4_top` 补充票，这是进入候选池后的风险质量参考；对 pattern 命中票，P1/P2 低分不自动剔除，但必须降低仓位或等待更清晰的回撤/确认。
- P1/P2 推荐二次门槛按来源分层：
  - `phase4_top` 补充票：代码已要求 P1 `>= 40`、P2 `>= 50`、P4 `>= 70`；最终写入时优先看 P1/P2 是否接近 `80`，而不是单纯越高越好。
  - pattern 命中票：P1/P2 不设硬底线，但 P4 必须 `> 70`；如果结构完整、主线匹配、止损清楚，即使风险分偏低也可以保留到最终候选讨论。
  - 非 pattern 来源只要 P1 或 P2 低于自动门槛，默认不进入最终选股结果表。
- `phase7_trade_permission`：这是日期级买点环境提示；如果为 `no_trade`，当天候选只按观察池写。
- P1/P2 接近 `80`：加分；明显偏低或过高都需要解释。
- `phase4_score_100` 更高：同等主线和同等风险条件下加分。对 `phase4_top` 补充票，Phase4 是收益排序主轴；对 pattern 命中票，Phase4 是入池硬门槛，必须 `> 70`。
- `phase5_score_100` 偏低：降级，并在推荐理由里提示长周期极端风险。
- `macd_bottom_divergence_15d = true`、`macd_cross_state = golden_cross`、`bullish_volume_price_divergence_flag = true`：加分。
- `macd_top_divergence_15d = true`、`macd_cross_state = dead_cross`、`bearish_volume_price_divergence_flag = true`：降级，通常放到“值得注意的股”或写风险。
- `ATR%` 过高、`2ATR` 止损离当前价太远、`建议总仓位%` 明显偏低：降级，因为止损成本过大且可承受仓位有限。

模式1/2/3自动筛选：

### 4. 操作顺序

实际写入时，建议按下面顺序执行：

1. 先确认主 `watchlist` 已经来自同日 Phase1/Phase2 硬过滤后的结果

### 5. 底背离偏机会，顶背离偏风险

在“值得注意的股”里，背离信号要有明确倾向：

- 底背离：优先提示“修复机会”或“值得重点观察”
- 顶背离：优先提示“冲高风险”或“不宜盲目追高”
- `MACD` 金叉：优先提示“趋势修复”或“有继续走强可能”
- `MACD` 死叉：优先提示“转弱风险”或“节奏需放慢”
- `量价看多`：优先提示“量价配合改善”
- `量价看空`：优先提示“量价不配合，冲高持续性存疑”

不要把两者写成同样口径。
