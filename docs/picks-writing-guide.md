# 选股.md 写入指导

本文用于统一后续向 [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 写入内容时的格式、顺序与分析口径。

目标不是让 `选股.md` 变成技术结果原始转储，而是让它成为“基于当日 `watchlist_pattern` / `watchlist_trend`、结合主线与主观判断后的最终整理记录”。

## 使用范围

本指导适用于所有需要新增或更新 [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 的场景。

写入时必须遵守以下前提：

1. 基于当日 `watchlist_pattern` 和 `watchlist_trend` 进行选股
2. 结合 [主线.md](/C:/Users/wdyab/Desktop/wdy/stocks/主线.md)，优先选择博主给出的主线对应个股
3. 新日期的结果写在文件最前，旧日期的结果保留在文件后面
4. 一个日期内，先给出最终选股结果，再给出简要分析

## 总体原则

### 1. `选股.md` 不是候选池原样转储

`watchlist_pattern` 和 `watchlist_trend` 是技术候选池，`选股.md` 是基于候选池进一步整理后的结果。

因此写入时：

- 不要求把全部候选都写进去
- 可以只保留少量更符合主线和当日判断的标的
- 不要把纯技术强度但明显脱离当前主线的个股全部塞进最终记录

### 2. 主线优先

选股时要先看 [主线.md](/C:/Users/wdyab/Desktop/wdy/stocks/主线.md)。

优先顺序建议如下：

1. 博主明确指出的第一梯队主线
2. 明确指出的第二梯队主线
3. 轮动与预期线
4. 短线事件题材
5. 无法归入主线、只有技术形态支撑的个股

如果某只股票技术面很好，但不属于当前主线，应降低优先级。

### 3. 结果先于分析

一个日期段落内部，必须先给出选股结果，再给出分析。

顺序固定为：

1. 日期标题
2. 选股结果表格
3. 当日市场情绪
4. 主线变更
5. 值得注意的股
6. 如有必要，再补充简短备注

## 文件排序规则

`选股.md` 中日期必须按“新到旧”排序。

也就是：

- 最新日期写在文件最前面
- 旧日期保留在后面
- 更新某一天内容时，只替换该日期对应段落，不要打乱其他日期顺序

示意：

```md
### 2026.4.15

...

### 2026.4.14

...

### 2026.4.13

...
```


## 表格写法要求

表格字段固定如下：

- `梯队`
- `来源`
- `股票代码`
- `股票名称`
- `行业/主线`
- `符合模式/背离`
- `五日分数`
- `五日均分`
- `TradingView标签`
- `BuyScore`
- `PriceAction`
- `趋势/触发补充`
- `推荐理由`

### 字段解释

`梯队`

- 表示你最终整理后的优先级
- 一般可写 `第一梯队`、`第二梯队`
- 如果没有必要，不建议在最终记录里保留太多 `第三梯队`

`来源`

- 统一写 `双命中`、`pattern`、`trend`
- `双命中` 指同时出现在 `watchlist_pattern` 和 `watchlist_trend`
- 如果只命中 `pattern`，但同时 `in_trend_universe = true`，可以优先保留

`行业/主线`

- 优先填写与 [主线.md](/C:/Users/wdyab/Desktop/wdy/stocks/主线.md) 对应的主线名称
- 如果确实无法归类，再写 `未分类`

`符合模式/背离`

- 写清 pattern 编号
- 如果存在顶背离、底背离、MACD 背离、量价背离，要明确写出来
- 示例：
  - `pattern 1`
  - `pattern 2 + 底背离`
  - `pattern 1 + 顶背离`
  - `pattern 3 + MACD底背离 + 量价看多`

`BuyScore`

- 对应 `watchlist` 中的趋势复核总分
- 用来表达这只票在新趋势体系下的综合通过程度
- 一般不需要写太多小数，保留 1 位到 2 位即可

`PriceAction`

- 对应 `price_action_score`
- 主要反映突破/回踩形态本身的结构质量
- 当两只股票都属于同一主线时，可以把它作为重要比较项

`趋势/触发补充`

- 用来简短写出趋势复核里的新增状态字段
- 优先写这些内容：
  - `signal_type`，如 `breakout / pullback`
  - `macd_cross_state`，如 `金叉 / 死叉 / 上方运行 / 下方运行`
  - `macd_divergence_state`，如 `MACD顶背离 / MACD底背离`
  - `volume_price_divergence_state`，如 `量价看多 / 量价看空`
- 这一列是“辅助判断”，不要求把全部字段原样抄进去
- 示例：
  - `breakout + 金叉`
  - `pullback + MACD底背离`
  - `breakout + 量价看多`

`推荐理由`

- 必须是简短判断，不要照搬原始技术字段
- 推荐理由应体现：
  - 是否属于主线
  - 技术形态是否完整
  - 是否通过趋势复核
  - 是否有背离或金叉死叉信号
  - 是否适合重点关注或降低预期

## 选股规则

### 1. 必须基于当日候选池

写入 [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 时，候选来源必须来自当日 `watchlist_pattern` / `watchlist_trend`，而不是随意从全市场结果中重挑。

换句话说：

- 先看 `watchlist_pattern`
- 再看 `watchlist_trend`
- 双命中的票必须优先推荐
- 主线内的 `trend-only` 票应尽量进主表展示
- `pattern-only` 的票，优先保留 `in_trend_universe = true` 的
- 不要跳过这两份候选池直接从全市场结果选股

### 2. 主线优先于纯技术

当主线和技术面冲突时，优先考虑主线一致性。

推荐选择逻辑：

1. 先找属于当日主线的双命中个股
2. 再补主线里的 `trend-only`
3. 再补主线里的 `pattern-only`
4. `pattern-only` 里优先看 `in_trend_universe = true`
5. 如果同主线差不多，再比较 `buy_score`、`price_action_score` 和 `趋势/触发补充`
6. 最后才考虑非主线但技术特别突出的个股作为补充

### 3. 操作顺序

实际写入时，建议按下面顺序执行，避免把 `trend-only` 压得过后：

1. 先把 `双命中` 的票挑出来，这部分必须优先推荐
2. 再看 `trend-only`，只要属于当前主线，尽量进主表
3. 再看 `pattern-only`，其中 `in_trend_universe = true` 的优先级更高
4. 如果主表数量已经够了，主线外的 `trend-only / pattern-only` 只放到“值得注意的股”
5. 如果主线内候选很少，可以允许主线外但结构特别完整的票补位

### 4. 底背离偏机会，顶背离偏风险

在“值得注意的股”里，背离信号要有明确倾向：

- 底背离：优先提示“修复机会”或“值得重点观察”
- 顶背离：优先提示“冲高风险”或“不宜盲目追高”
- `MACD` 金叉：优先提示“趋势修复”或“有继续走强可能”
- `MACD` 死叉：优先提示“转弱风险”或“节奏需放慢”
- `量价看多`：优先提示“量价配合改善”
- `量价看空`：优先提示“量价不配合，冲高持续性存疑”

不要把两者写成同样口径。

## 分析部分的写法要求

### 1. 当日市场情绪监测

这一段要回答：

- 今天整体是强、弱，还是强中有分化
- 候选股是否集中
- 是否存在明显风险信号

写法建议：

- 用一句到两句短句
- 不要写成长篇复盘
- 重点说明“强弱”和“风险”

示例：

```md
当日市场情绪监测：候选股数量中等，强势票主要集中在算力硬件和电池方向，整体情绪偏强但高位顶背离个股开始增多，追高需要控制节奏。
```

### 2. 主线变更

这一段要回答：

- 当前主线是否延续
- 哪些方向在强化
- 哪些方向在转弱或轮动

写法建议：

- 明确点出主线名称
- 如果发生切换，要直说“主线切换”
- 如果只是延续，要直说“主线延续”

示例：

```md
主线变动：主线仍然围绕算力硬件和电池展开，创新药保持次主线地位，机器人更偏轮动补涨，稳定币/支付暂时只适合事件性观察。
```

### 3. 值得注意的股

这一段要逐只点名少量关键股票。

建议只写 3 到 5 只，不要把整张表再重复一遍。

优先写这些情况：

- 属于主线且技术强度很高
- 出现底背离，值得重点观察
- 出现顶背离，需要提示风险
- `buy_score / price_action_score` 在同主线里明显更高
- 出现金叉、量价看多，说明趋势触发质量更好
- 连续多日出现在候选池中，说明强度延续

示例：

```md
值得注意的股：002579中京电子出现底背离且 `buy_score`、`price_action_score` 都处在较高水平，若量能继续修复可重点跟踪；601000唐山港出现顶背离且死叉压力开始出现，若继续冲高不宜追价；603906龙蟠科技连续多日保持强势，并伴随量价看多，属于主线中的核心观察标的。
```

## 推荐的单日写入流程

每次更新 [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 时，建议按以下顺序执行：

1. 读取当日 `watchlist`
2. 对照 [主线.md](/C:/Users/wdyab/Desktop/wdy/stocks/主线.md) 标记主线归属
3. 从 `watchlist` 中选出更符合主线的个股
4. 按优先级整理表格
5. 写“市场情绪监测”
6. 写“主线变动”
7. 写“值得注意的股”
8. 将新日期段落插入文件最前

## 禁止事项

写入时不要这样做：

- 不要把整个 `watchlist` 原样全部贴进 `选股.md`
- 不要跳过 [主线.md](/C:/Users/wdyab/Desktop/wdy/stocks/主线.md) 只按技术强度排序
- 不要把新日期写到旧日期后面
- 不要在一个日期段落里先写分析、后写结果
- 不要对底背离和顶背离给出同样口径
- 不要把最终判断写成过长的段落，保持简洁

## 最终目标

[选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 应该体现的是：

- 候选来源有据可依
- 主线判断优先
- 结构统一，便于长期复盘
- 每个日期都能快速看出“选了什么、为什么选、风险在哪里”
