# 选股.md 写入指导

本文用于统一后续向 [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 写入内容时的格式、顺序与分析口径。

目标不是让 `选股.md` 变成技术结果原始转储，而是让它成为“基于当日 `watchlist_pattern` / `watchlist_trend`、结合主线与主观判断后的最终整理记录”。

## 使用范围

本指导适用于所有需要新增或更新 [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 的场景。

写入时必须遵守以下前提：

1. 基于当日 `watchlist_pattern` 和 `watchlist_trend` 进行选股
2. 结合 [主线.md](/C:/Users/wdyab/Desktop/wdy/stocks/主线.md)，优先选择博主给出的主线对应个股
3. 新日期的结果写在文件最前，旧日期的结果保留在文件后面
4. 一个日期内，先给出最终选股结果表，再给出交易辅助信息表，最后给出简要分析

另外，写入表格时应尽量排除明显英文状态值：

- `模式`、`买入`、`强力买入`、`突破`、`回踩` 等优先使用中文
- `TradingView`、`ATR14`、`ATR%`、`MACD` 等通用缩写可保留

## 总体原则

### 1. `选股.md` 不是候选池原样转储

`watchlist_pattern` 和 `watchlist_trend` 是技术候选池，`选股.md` 是基于候选池进一步整理后的结果。

因此写入时：

- 不要求把全部候选都写进去
- 可以只保留少量更符合主线和当日判断的标的
- 不要把纯技术强度但明显脱离当前主线的个股全部塞进最终记录

### 2. 主线优先

选股时要先看 [主线.md](/C:/Users/wdyab/Desktop/wdy/stocks/主线.md)。

优先顺序建议如下：

1. 博主明确指出的第一梯队主线
2. 明确指出的第二梯队主线
3. 轮动与预期线
4. 短线事件题材
5. 无法归入主线、只有技术形态支撑的个股

如果某只股票技术面很好，但不属于当前主线，应降低优先级。

### 3. 结果先于分析

一个日期段落内部，必须先给出选股结果，再给出分析。

顺序固定为：

1. 日期标题
2. 最终选股结果表
3. 交易辅助信息表
4. 逐股口语化说明
5. 当日市场情绪
6. 主线变更
7. 值得注意的股
8. 如有必要，再补充简短备注

## 文件排序规则

`选股.md` 中日期必须按“新到旧”排序。

也就是：

- 最新日期写在文件最前面
- 旧日期保留在后面
- 更新某一天内容时，只替换该日期对应段落，不要打乱其他日期顺序

示意：

```md
### 2026.4.15

...

### 2026.4.14

...

### 2026.4.13

...
```

```md
| 股票代码 | 股票名称 | ATR14 | ATR% | 建议止损（2倍ATR） | 建议止盈（2倍ATR） | BuyScore | PriceAction | 趋势/触发补充 |

逐股说明：

- 000000示例股票：这是 `模式2`，可写成“4 月 12 日放量突破 3 月 8 日前高 18.60，当前仍在突破位上方不远，量能放大约 `3.2` 倍，次日重点看能否继续站稳突破位”。若是 `模式1`，则写清“接近哪一天前高、还差多少”；若是 `模式3`，则写清“突破后第几天、离前高上方多少、是否回踩 `MA20`”；并结合 `ATR` 止损止盈价提示注意点位。

当日市场情绪监测：一句话总结市场强弱、候选数量、强度分布和风险点。

值得注意的股：点名主表之外或不宜直接放进逐股说明的关键个股，底背离优先提示机会，顶背离优先提示风险；如有必要，可同时点出 `buy_score`、`price_action_score`、金叉死叉和量价背离状态。

```

## 表格写法要求

表格拆成两个。

最终选股结果表字段固定如下：

- `股票代码`
- `股票名称`
- `行业/主线`
- `符合模式/背离`
- `五日均分`
- `TradingView标签`
- `连续上榜天数`
- `推荐理由`

交易辅助信息表字段固定如下：

- `股票代码`
- `股票名称`
- `ATR14`
- `ATR%`
- `建议止损（2倍ATR）`
- `建议止盈（2倍ATR）`
- `BuyScore`
- `PriceAction`
- `趋势/触发补充`

### 字段解释

`优先级与来源`

- 最终选股结果表不再单列 `梯队` 和 `来源`
- 优先级通过主表从上到下的排序体现，越靠上优先级越高
- `双命中 / 仅趋势命中 / 仅模式命中` 仍然要参与判断，但可写进 `推荐理由` 或后文分析，不再单独占一列

`连续上榜天数`

- 指截止当日，该股票连续出现在主 `watchlist_YYYY-MM-DD.json` 的天数
- 必须按主 `watchlist` 计算，不按 `watchlist_pattern` 或 `watchlist_trend` 单独计算
- 中间只要断一天，下一次重新从 `1` 开始
- 这是持续强度参考，不单独决定是否推荐，但同等条件下连续天数更高的票可优先

`行业/主线`

- 优先填写与 [主线.md](/C:/Users/wdyab/Desktop/wdy/stocks/主线.md) 对应的主线名称
- 如果确实无法归类，再写 `未分类`

`符合模式/背离`

- 写清模式编号
- 如果存在顶背离、底背离、MACD 背离、量价背离，要明确写出来
- 示例：
  - `模式1`
  - `模式2 + 底背离`
  - `模式1 + 顶背离`
  - `模式3 + MACD底背离 + 量价看多`

### 模式 1-6 的具体写法要求

不能只写“模式1 / 模式2 / 模式3 ...”，必须尽量解释成具体价格结构。优先使用 `watchlist` 中已经带出的模式字段；如果仍不够，再回看同日 `patterns_all_YYYY-MM-DD.csv`。

优先引用这些字段：

- `old_high_date`
- `old_high_price`
- `distance_to_old_high_pct`
- `breakout_date`
- `breakout_volume_ratio`
- `days_after_breakout`
- `platform_window_days`
- `platform_range_pct`
- `distance_to_platform_high_pct`
- `distance_to_ma20`
- `consolidation_days`
- `consolidation_range_pct`

推荐固定句式如下：

- `模式1`
  写成“接近某日前高”的口径。
  示例：`接近 3 月 18 日前高 24.60，当前距离前高约 4.2%，属于临门一脚型，次日重点看是否放量突破。`

- `模式2`
  写成“某日放量突破某日前高”的口径。
  示例：`4 月 20 日放量突破 2 月 27 日前高 18.30，成交量约为近 20 日均量的 3.4 倍，属于刚启动型，次日看是否继续站稳突破位。`

- `模式3`
  写成“突破后第 N 天”的口径。
  示例：`4 月 17 日突破后第 3 天，当前仍只比前高高 5.1%，若回踩 MA20 但收盘能站回去，仍可视作二次上车窗口。`

- `模式4`
  写成“突破震荡平台”的口径。
  示例：`4 月 20 日突破近 20 日震荡平台，平台振幅约 8.5%，当前只比平台上沿高 3.2%，属于平台突破后不远的位置。`

- `模式5`
  写成“趋势回踩”的口径。
  示例：`趋势回踩 MA20，当前距离 MA20 约 1.8%，若回踩缩量且重新转强，更适合按趋势延续理解。`

- `模式6`
  写成“整理后再启动”的口径。
  示例：`前面整理了 7 天，整理区间约 6.2%，当前重新向上试图走二波，重点看整理上沿附近是否能放量站稳。`

写逐股说明时，至少包含下面 4 类信息：

1. 它是哪个模式，对应的实际形态是什么
2. 关键锚点日期或平台，例如前高日期、突破日期、平台区间
3. 现在离关键位还有多远，或者已经高出多少
4. 次日最该注意的点位，例如前高位、平台上沿、`MA20`、`1ATR/2ATR` 止损位

如果字段缺失，不要编造；可以退回到较保守的写法，例如：

- `接近前高，次日看是否放量突破`
- `刚突破平台，次日看是否站稳平台上沿`
- `突破后延续，注意 MA20 附近承接`

`BuyScore`

- 对应 `watchlist` 中的趋势复核总分
- 用来表达这只票在新趋势体系下的综合通过程度
- 一般不需要写太多小数，保留 1 位到 2 位即可

`PriceAction`

- 对应 `price_action_score`
- 主要反映突破/回踩形态本身的结构质量
- 当两只股票都属于同一主线时，可以把它作为重要比较项

`ATR14`

- 对应 `watchlist` 中的 `ATR14`
- 用于描述个股的日线平均波动幅度
- 建议保留 1 位到 2 位小数

`ATR%`

- 对应 `watchlist` 中的 `ATR%`
- 用于横向比较不同股票的相对波动率
- 必须写成百分比形式

`建议止损`

- 必须填写
- 固定写 `2倍ATR` 对应的具体价格，不再加 `2ATR止损参考` 这类文字前缀
- 示例：`18.30`

`建议止盈`

- 必须填写
- 固定写 `2倍ATR` 对应的具体价格，不再加 `2ATR止盈参考` 这类文字前缀
- 示例：`21.70`

`趋势/触发补充`

- 用来简短写出趋势复核里的新增状态字段
- 优先写这些内容：
  - `signal_type`，如 `突破 / 回踩`
  - `macd_cross_state`，如 `金叉 / 死叉 / 上方运行 / 下方运行`
  - `macd_divergence_state`，如 `MACD顶背离 / MACD底背离`
  - `volume_price_divergence_state`，如 `量价看多 / 量价看空`
- 这一列是“辅助判断”，不要求把全部字段原样抄进去
- 示例：
  - `突破 + 金叉`
  - `回踩 + MACD底背离`
  - `突破 + 量价看多`

### 明显英文状态值的推荐中文写法

- `pattern 1 / 2 / 3 / ...` -> `模式1 / 模式2 / 模式3 / ...`
- `buy` -> `买入`
- `strong_buy` -> `强力买入`
- `breakout` -> `突破`
- `pullback` -> `回踩`
- `pattern-only` -> `仅模式命中`
- `trend-only` -> `仅趋势命中`

`推荐理由`

- 必须是简短判断，不要照搬原始技术字段
- 推荐理由应体现：
  - 是否属于主线
  - 技术形态是否完整
  - 是否通过趋势复核
  - 是否有背离或金叉死叉信号
  - 是否适合重点关注或降低预期

## 选股规则

### 1. 必须基于当日候选池

写入 [选股.md](/C:/Users/wdyab/Desktop/wdy/stocks/选股.md) 时，候选来源必须来自当日 `watchlist_pattern` / `watchlist_trend`，而不是随意从全市场结果中重挑。

换句话说：

- 先看 `watchlist_pattern`
- 再看 `watchlist_trend`
- 双命中的票必须优先推荐
- 主线内的 `仅趋势命中` 票应尽量进主表展示
- `仅模式命中` 的票，优先保留 `in_trend_universe = true` 的
- 不要跳过这两份候选池直接从全市场结果选股

### 2. 主线优先于纯技术

当主线和技术面冲突时，优先考虑主线一致性。

推荐选择逻辑：

1. 先找属于当日主线的双命中个股
2. 再补主线里的 `仅趋势命中`
3. 再补主线里的 `仅模式命中`
4. `仅模式命中` 里优先看 `in_trend_universe = true`
5. 如果同主线差不多，再比较 `buy_score`、`price_action_score` 和 `趋势/触发补充`
6. 最后才考虑非主线但技术特别突出的个股作为补充

### 3. 操作顺序

实际写入时，建议按下面顺序执行，避免把 `仅趋势命中` 压得过后：

1. 先把 `双命中` 的票挑出来，这部分必须优先推荐
2. 再看 `仅趋势命中`，只要属于当前主线，尽量进最终选股结果表
3. 再看 `仅模式命中`，其中 `in_trend_universe = true` 的优先级更高
4. 如果主表数量已经够了，主线外的 `仅趋势命中 / 仅模式命中` 只放到“值得注意的股”
5. 如果主线内候选很少，可以允许主线外但结构特别完整的票补位
6. 进入最终选股结果表的股票，必须同步出现在“交易辅助信息表”里，并补全 ATR、建议止损、建议止盈

### 4. 底背离偏机会，顶背离偏风险

在“值得注意的股”里，背离信号要有明确倾向：

- 底背离：优先提示“修复机会”或“值得重点观察”
- 顶背离：优先提示“冲高风险”或“不宜盲目追高”
- `MACD` 金叉：优先提示“趋势修复”或“有继续走强可能”
- `MACD` 死叉：优先提示“转弱风险”或“节奏需放慢”
- `量价看多`：优先提示“量价配合改善”
- `量价看空`：优先提示“量价不配合，冲高持续性存疑”

不要把两者写成同样口径。

### 5. 后续新日期必须加入 ATR 与交易参考信息

从本指导更新之后，所有新增日期都必须：

1. 使用“双表结构”
2. 在交易辅助信息表中写入 `ATR14` 和 `ATR%`
3. 写入 `建议止损（2倍ATR）`
4. 写入 `建议止盈（2倍ATR）`
5. 在最终选股结果表中写入 `连续上榜天数`

旧日期不强制回填，但后续新日期不得省略这些字段。

## 分析部分的写法要求

### 0. 逐股口语化说明

这一段是主表之后最重要的正文，不能只重复表格字段，也不能只写一句空泛判断。

写法要求：

- 按主表顺序逐只写
- 用自然语言，不要写成机器字段堆砌
- 每只票尽量写成 1 到 3 句
- 优先解释“它为什么是这个模式”
- 必须带关键点位或关键结构，至少包含前高、平台、`MA20`、`ATR` 止损位中的一项

建议句式：

- `002579中京电子是模式1，当前已经接近 3 月 18 日前高 24.60，距离前高只差约 4%，属于临门一脚型。次日重点看能否放量过前高，若迟迟不过，前高附近反而容易形成短压。`
- `600184光电股份是模式4，属于平台突破后不远的位置，4 月 20 日刚突破近一段时间的震荡上沿，当前还没有离平台太远。只要平台上沿不被快速跌回，短线仍可按突破延续看。`
- `001287中电港如果按模式3理解，就是突破后延续阶段，当前仍在前高上方不远，同时又带 MACD 底背离修复。要注意的是如果回踩 MA20 后收不回去，二次上车逻辑就会变弱。`

### 1. 当日市场情绪监测

这一段要回答：

- 今天整体是强、弱，还是强中有分化
- 候选股是否集中
- 是否存在明显风险信号

写法建议：

- 用一句到两句短句
- 不要写成长篇复盘
- 重点说明“强弱”和“风险”

示例：

```md
当日市场情绪监测：候选股数量中等，强势票主要集中在算力硬件和电池方向，整体情绪偏强但高位顶背离个股开始增多，追高需要控制节奏。
```

### 2. 主线变更

这一段要回答：

- 当前主线是否延续
- 哪些方向在强化
- 哪些方向在转弱或轮动

写法建议：

- 明确点出主线名称
- 如果发生切换，要直说“主线切换”
- 如果只是延续，要直说“主线延续”

示例：

```md
主线变动：主线仍然围绕算力硬件和电池展开，创新药保持次主线地位，机器人更偏轮动补涨，稳定币/支付暂时只适合事件性观察。
```

### 3. 值得注意的股

这一段要逐只点名少量关键股票。

建议只写 3 到 5 只，不要把整张表再重复一遍。

优先写这些情况：

- 属于主线且技术强度很高
- 出现底背离，值得重点观察
- 出现顶背离，需要提示风险
- `buy_score / price_action_score` 在同主线里明显更高
- 出现金叉、量价看多，说明趋势触发质量更好
- 连续多日出现在候选池中，说明强度延续

示例：

```md
值得注意的股：002579中京电子出现底背离且 `buy_score`、`price_action_score` 都处在较高水平，若量能继续修复可重点跟踪；601000唐山港出现顶背离且死叉压力开始出现，若继续冲高不宜追价；603906龙蟠科技连续多日保持强势，并伴随量价看多，属于主线中的核心观察标的。
```

## 推荐的单日写入流程
