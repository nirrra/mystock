
## 核心指标

| 指标 | 含义 | 当前用途 |
|---|---|---|
| P1 | 全市场尾部风险模型，分数越高风险质量越好 | 个股风险过滤 |
| P2 | Triple-barrier 交易型风险模型，分数越高交易风险越低 | 个股风险过滤 |
| P4 | Qlib Alpha158 + LightGBM 收益回归模型，分数越高上涨质量越好 | 个股收益排序主轴 |
| P9 | 板块未来 20 日收盘上涨超过 5% 的概率模型 | 评价板块轮转和买入预期 |
| 长期主线指数 | 板块过去较长周期内是否持续受到资金关注 | 找长期主线板块 |
| 短期主线指数 | 板块最近一段时间是否正在带动市场 | 找当下活跃主线 |
| 龙头指数 | 个股在所属板块过去两年里的领涨能力 | 重建关切板块、筛板块龙头 |

## 关切板块

行业和概念原始映射噪声较大，因此在 `analyze-sector-leaders` 后会基于个股龙头指数重新整理“关切板块”：

```text
combined_leader_score >= 60
```

达到阈值才认为股票与该行业/概念有真实联系。行业和概念统一称为“关切板块”。没有任何关切板块的股票标记为“弱势股”。

## 三条路线

A1 近期强势：长期主线 Top100 内的短期强度 Top20；真实属于该关切板块；程序层面只做公共过滤 `涨幅<=9.9%、P1>20、P2>20`。混合分高、龙头指数高、P9 是否仍高，留给复盘阶段灵活筛选。

A2 轮转预期：长期主线 Top100 内的 P9 Top20；真实属于该关切板块；程序层面筛到 `涨幅<=9.9%、P1>40、P2>40` 后停止。混合分高、龙头指数高、P4 不弱、是否适合低位分批建仓，留给复盘阶段灵活筛选。

B 形态符合：命中任一 Pattern，且通过公共过滤 `涨幅<=9.9%、P1>20、P2>20`。B 不与 A1/A2 混排，只作为结构候选。若 B 同时贴近 A1/A2 主线，可以人工提高关注级别。

所有路线都排除当日涨幅 `>9.9%` 的股票。

## 交易纪律摘要

仓位按 ATR 反推。设总资金为 `E`，初始买入价为 `P`，`D = 2 * ATR14`，总最大亏损为 `2% * E`。当前分批方案在第三次加仓前的平均止损距离约为 `0.85D`：

```text
理论仓位比例 = 0.02 * P / (0.85 * D)
             = 0.01 / 0.85 * P / ATR14
```

实际仓位仍受单一标的最大 40% 限制。买入后 5 个交易日仍不能脱离成本区或站稳结构位，应减仓或退出。
