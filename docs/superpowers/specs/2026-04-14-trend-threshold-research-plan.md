# Trend Threshold Research Plan

## 1. 目标

为当前“收盘评分 -> 次日开盘买入”的趋势交易链路建立一套可复用的阈值研究方法，用历史样本反推更合理的买入阈值，而不是直接拍脑袋设定固定分数。

本计划的目标不是立刻修改交易规则，而是先回答下面几个问题：

- 哪些评分分项在强势样本和弱势样本之间有明显分布差异
- `buy_score`、`price_action_score`、`macd_score` 等分项的合理阈值大概落在哪些区间
- `breakout` 和 `pullback` 是否应该共用同一套阈值
- 不同持有周期下，阈值是否稳定

## 2. 核心原则

### 2.1 先固定初筛层，再研究阈值

阈值研究不能把待研究的门槛提前混入样本筛选。

研究样本只保留基础结构条件：

- 已进入 `trend universe`
- 已识别出 `breakout` 或 `pullback` setup
- 信号日之后存在次日开盘价，满足可交易条件

在这一步，不使用 `buy_score >= X`、`price_action_score >= Y` 之类的过滤条件。

### 2.2 先打标签，再比较分布

阈值不应直接从“整体胜率”倒推，而应先根据未来表现给样本打标签，再比较不同标签组的指标分布。

正确顺序应为：

`样本抽取 -> 次日开盘回测 -> 未来收益打标签 -> 指标分布比较 -> 生成阈值候选 -> 回测验证`

### 2.3 多时间截面、多持有周期、样本外验证

阈值如果只在单一时段有效，结论没有意义。

阈值研究至少要覆盖：

- 多个历史时间截面
- 多个持有周期
- 样本内研究与样本外验证

## 3. 样本构建

### 3.1 研究区间

建议研究区间覆盖完整的不同市场状态，例如：

- `2024-01-01` 到 `2026-04-10`

### 3.2 时间截面抽样

不要只选一个日期。建议按月或按周选取多个历史时间点，例如：

- 每月选 `1` 到 `2` 个交易日
- 或每周固定选一个交易日

每个截面都执行同样的“趋势初筛 + setup 识别 + 收盘评分”流程，得到当日全部候选样本。

### 3.3 样本字段

每条样本至少保留以下字段：

- `trade_date`
- `planned_entry_date`
- `symbol`
- `setup_type`
- `trend_score`
- `entry_score`
- `trend_base_score`
- `price_action_score`
- `macd_score`
- `volume_score`
- `volume_price_divergence_score`
- `boll_score`
- `rsi_score`
- `kdj_score`
- `atr_score`
- `buy_score`
- `positive_indicator_count`

## 4. 标签定义

### 4.1 执行口径

统一使用当前更接近真实执行的研究口径：

- `t` 日收盘后生成评分
- `t+1` 日开盘买入

### 4.2 未来表现标签

对每条样本分别计算未来持有 `5/10/20/40` 个交易日的表现：

- 最终收益 `return_pct`
- 最大回撤 `max_drawdown_pct`
- 最大上涨 `max_upside_pct`

### 4.3 样本分组

建议至少形成以下标签组：

- 强势组：未来收益前 `20%`，或未来收益高于预设高收益门槛
- 弱势组：未来收益 `< 0`
- 可选补充组：大亏组，即未来收益后 `20%`

其中“前 `20%`”和“后 `20%`”的分位法通常比手写绝对涨幅门槛更稳，因为不同阶段市场波动不一样。

## 5. 指标分布比较

### 5.1 比较对象

重点比较强势组和弱势组在以下指标上的分布差异：

- `buy_score`
- `trend_base_score`
- `price_action_score`
- `macd_score`
- `volume_price_divergence_score`
- `positive_indicator_count`
- 其他单项评分

### 5.2 分布统计

每个指标至少输出：

- 样本数
- 均值
- 中位数
- `P20/P40/P50/P60/P80`
- 强势组与弱势组的重叠区间

### 5.3 研究目标

重点不是“哪个分数平均更高”，而是：

- 哪些指标对强弱样本有稳定区分度
- 合理的分界点大概落在哪
- 该指标是否值得作为硬门槛，而不是只作为加权项

## 6. 阈值候选生成

在指标分布比较完成后，为每个重点指标生成候选阈值，例如：

- `buy_score >= 75 / 80 / 85`
- `trend_base_score >= 60 / 65 / 70`
- `price_action_score >= 55 / 60 / 65`
- `macd_score >= 30 / 35 / 40`
- `positive_indicator_count >= 3 / 4 / 5`

候选阈值应来源于样本分位数和强弱组分界，而不是主观填写。

## 7. 阈值回测验证

### 7.1 单指标验证

分别验证每个候选阈值对下列结果的影响：

- 样本量
- 胜率
- 平均收益
- 中位数收益
- 平均最大回撤

### 7.2 组合阈值验证

进一步验证组合过滤条件，例如：

- `buy_score >= 80 and price_action_score >= 60`
- `buy_score >= 80 and macd_score >= 35`
- `buy_score >= 80 and positive_indicator_count >= 3`

### 7.3 按 setup 类型验证

必须分别验证：

- `breakout`
- `pullback`

这两类 setup 很可能不应该共用完全相同的门槛。

### 7.4 按持有周期验证

必须分别验证：

- `5` 日
- `10` 日
- `20` 日
- `40` 日

因为适合短波段的阈值，不一定适合更长持有周期。

## 8. 样本外验证

建议把研究区间拆成两段：

- 样本内：用于发现分布规律和生成候选阈值
- 样本外：用于验证候选阈值是否仍然有效

示例：

- 样本内：`2024-01-01` 到 `2025-06-30`
- 样本外：`2025-07-01` 到 `2026-04-10`

如果某个阈值只在样本内有效、样本外失效，就不能作为默认门槛。

## 9. 输出结果

本研究阶段应至少产出以下结果文件：

- 历史样本明细
- 强势组与弱势组指标分布对比表
- 单指标候选阈值表
- 组合阈值回测对比表
- `breakout` 与 `pullback` 分开统计表
- 样本内 / 样本外验证结果表

## 10. 最终落地方式

本研究的最终目标不是只写一份分析表，而是把结论收敛为三类可执行规则：

- 默认全局阈值
- `breakout` 专用阈值
- `pullback` 专用阈值

在进入代码默认配置前，这些阈值应先经过：

- 分布分析
- 单指标验证
- 组合验证
- 样本外验证

## 11. 当前建议

在正式做阈值研究之前，先保持当前默认门槛作为临时研究配置：

- `buy_score >= 80`
- `trend_base_score >= 65`
- `price_action_score >= 60`
- `macd_score >= 35`
- `positive_indicator_count >= 3`

这组值只作为第一版研究默认值，不视为最终交易规则。
